＃ 도커（Docker)

##  도커란?

- 도커는 애플리케이션 신속하게 구축, 테스트 및 배포할 수 있는 소프트웨어 플랫폼
- 애플리케이션을 컨테이너 단위로 묶어서, 어느 환경에서든 실행될 수 있게 만들어주는 플랫폼

---

## 컨테이너, 이미지

### 컨테이너

#### 컨테이너 개념

```text
[google 정의] 컨테이너는 소프트웨어 서비스를 실행하는 데 필요한 특정 버전의 프로그래밍 언어 런타임 및 라이브러리와 같은 종속 항목과 애플리케이션 코드를 함께 포함하는 경량 패키지
[AWS 정의] 컨테이너는 애플리케이션 코드, 라이브러리 및 클라우드에서 실행하기 위해 필요한 기타 종속성이 포함된 소프트웨어 코드 패키지
```

#### 컨테이너가 왜 필요한가

- **경량화 및 효율성** : 컨테이너는 기존 **가상 머신(VM)보다 훨씬 가볍고 빠르며**, 더 적은 자원으로 많은 애플리케이션을 운영할 수 있어 **리소스 활용도를 향상**하고 **서버 비용을 절감**할 수 있다.
- **이식성 및 일관성** : 컨테이너는 애플리케이션과 모든 종속성을 패키징하므로 **개발, 테스트, 생산 환경 간의 차이를 제거**하여 Linux, Windows, Mac 등 **운영체제에 구애받지 않고** 물리적 서버, 개발자 컴퓨터, 데이터센터, 퍼블릭 클라우드 등 사실상 **어느 환경에서나 구동 가능**하도록 함
- **빠른 배포 및 확상성** : **컨테이너화된 애플리케이션은 기존 방식보다 빠르게 배포**되며, 밀리초 단위로 시작하여 수초 내 확장이 가능하여 사용자 요청 대응 시간을 향상합니다. 이를 통해 애플리케이션 출시 속도가 향상되고 운영 위험이 감소
- **DevOps 및 CI/CD 용이성** : 컨테이너는 지속적 통합(CI) 및 지속적 배포(CD) 파이프라인과 연동하여 **코드 변경 사항이 자동으로 테스트되고 운영 환경에 안전하게 배포될 수 있도록 지원** 함
- 책임 분리 : 컨테이너화를 통해 개발자는 애플리케이션 로직과 종속성에 집중하고, IT 운영팀은 배포 및 관리에 집중하는 등 책임을 명확히 분리 가능
- **애플리케이션 격리 및 보안** : 각 컨테이너는 **독립적으로 실행**되어 문제 발생 시 영향을 격리하며, **보안 취약점 노출 범위를 감소**시킬 수 있습니다. 또한, 승인된 사용자와 관리자가 컨테이너 이미지를 서명할 수 있는 인프라가 통합되어 있어 **신뢰할 수 없는 컨테이너 배포를 방지**함

#### 클라우드에서 컨테이너 활용

- 클라우드의 핵심은 필요할 때 자원을 늘리고, 필요 없으면 줄이는 것이다. 컨테이너는 가볍고 빠르게 실행되는 특성 덕분에 이러한 클라우드 환경에 적합함

##### 마이크로서비스

- 하나의 애플리케이션을 기능 단위로 분리하여 각각 독립된 컨테이너로 실행하는 방식이다. 예를 들어 쇼핑몰의 경우 로그인, 상품, 결제, 배송 기능을 각각 별도의 컨테이너로 운영함
- 특정 기능에 장애가 발생해도 다른 기능은 정상적으로 동작함
- 특정 기능만 독립적으로 수정 및 배포가 가능함

---

##### 오토스케일링

- 트래픽에 따라 컨테이너 수를 자동으로 늘리고 줄이는 기능
- 트래픽 증가 시 컨테이너를 자동으로 추가 생성하여 대응함
- 트래픽 감소 시 컨테이너 수를 줄여 불필요한 비용을 절감함
- VM은 생성에 몇 분이 소요되지만, 컨테이너는 몇 초면 생성 가능하여 빠른 대응이 가능함

--- 

##### CI/CD 파이프라인

- 코드 변경 시 빌드, 테스트, 배포가 자동으로 이어지는 흐름이다.

1. 개발자가 코드를 Git에 push
2. 이미지 자동 빌드 (`docker build`)
3. 테스트 컨테이너에서 자동 테스트 수행
4. 운영 서버에 컨테이너 자동 배포

- 사람이 직접 배포 작업을 수행할 필요가 없어 human error를 줄일 수 있다.

##### 쿠버네티스

컨테이너 수가 많아질수록 관리가 어려워지는데, 이를 자동으로 관리해주는 도구이다.

| 기능 | 설명 |
|------|------|
| 자동 복구 | 컨테이너가 죽으면 자동으로 재실행 |
| 오토스케일링 | 트래픽에 따라 컨테이너 수 자동 조절 |
| 로드밸런싱 | 트래픽을 여러 컨테이너에 균등 분산 |
| 롤링 업데이트 | 서비스 중단 없이 새 버전으로 교체 |
