# LAN
- 좁은 지역 안에서 장치들을 연결한 네트워크

## 허브란 무엇인가

- 개념 : 허브는 네트워크에서 여러 장치를 하나로 연결해주는 중앙 집결 장치
- 개본 개념 : 여러 컴퓨터나 장치들을 하나의 네트워크로 묶어주는 연결 중심점
- 허브의 동작 방식
    1. 한포트에서 데이터가 들어오면
    2. 연결된 모든 포트로 그 데이터를 그냥 뿌려버린다.
    3. 받은 장치
  자기 것이 아니면 무시하는 방식
  - 즉 브로드캐스트 방식

## CSMA/CD

- (Carrier Sense Multiple Access/Collision Detection)
- 이더넷에서 여러 장치가 동시에 같은 회선을 사용할 때 충돌을 감지하고 제어하는 프로토콜
- 동작 순서
    1.Carrier Sense : 전송 전에 회선이 사용 중인지 먼저 확인
    2.Multiple Access : 여러 장치가 하나의 회선을 공유
    3.Collision Detection : 충돌이 발생하면 즉시 감지

### 동작 순서

```text
1.회선 감지 (CS)
- 회선이 비어있나 확인
2.비어잇으면 -> 데이터 전송 시작
3.전송 중 충돌 감지 (CD)
- 충돌 발생 시 -> JAM 신호 전송 (모두에게 충돌 알림)
4.랜덤 시간 대기 (Backoff)
- 각 장치가 서로 다른 랜덤 시간만큼 기다림
5.다시 1번부터 반복
```

### 충돌(Collision)

- 허브처럼 같은 회선을 공유하는 환경에서, 두 장치가 동시에 전송하면 신호가 겹처서 깨지는 현상
- 도메인(Collision Domain) : 충돌이 발생하는 영역

### Backoff 알고리즘

```text
- 충돌 후 대기 시간을 랜덤으로 정하는 이유는, 같은 시간에 또 재전송하면 또 충돌이 나기 때문이다.
  - 1번 충돌 -> 0~2슬롯 중 랜덤 대기
  - 2번 충돌 -> 0~4슬롯 중 랜덤 대기
  - 3번 충돌 -> 0~8슬롯 중 랜덤 대기
  - n번 충돌 -> 0~2^n 슬롯 중 랜덤 대기
  - 16번 초과 -> 전송 포기(에러 처리)
```

### CSMA/CD와 네트워크 장비의 관계

- 허브(Hub) : 모든 포트가 하나의 충돌 도메인 -> CSMA/CD 필수
- 스위치(Switch) : 포트마다 충돌 도메인 분리 -> 충돌 거의 없음
- 라우터(Router) : 완전히 분리
'스위치가 보편화되면서 CSMA/CD는 사실상 거의 사용되지 않지만, 네트워크의 근본 원리로 자주 나옴'

### CSMA/CD와의 차이점

- CSMA/CD
  - 의미 : 충돌 감지
  - 사용 환경 : 유선 이더넷
  - 이유 : 유선은 충돌 감지 가능
- CSMA/CA
  - (Carrier Sense Multiple Access with Collision Avoidance)
  - 의미 : 충돌 회피
  - 사용 환경 : 무선(Wi-Fi)
  - 이유 : 무선은 감지가 어려워 미리 회피

- 무선에서는 신호가 겹쳐도 자신이 보낸 건지 충돌인지 구분이 어렵기 때문에 CA 방식을 쓴다.

## MAC

- MAC(Media Access Control)
- 네트워크에서 장치를 식별하는 고유 주소 체계이자, 물리적 매체 접근을 제어하는 방식입니다.

### MAC의 두 가지 의미

1.MAC 주소 : 네트워크 장치에 부여된 고유 하드웨어 주소
2.MAC 프로토콜 : 공유 매체에 접근하는 방식

### MAC 주소란?

- 네트워크 인터페이스 카드(NIC)에 제조 시 영구적으로 부여되는 48비트 고유 식별자
  - 총 48비트 = 6바이트
  - 16진수 2자리씩 6묶음으로 표현
  - 구분자는 : or - 사용

### MAC 주소 구조

- 48비트 MAC 주소는 OUI(24비트)와 NIC 고유번호(24비트)로 구성된다
- OUI
  - Organizationally Unique Identifier
  - 제조사가 할당
- 같은 제조사라도 NIC 번호는 다르다.
- 이론상 전 세계에 동일한 MAC 주소는 없다.

### IP 주소 vs MAC 주소

#### MAC 주소

- 계층 : OSI 2계층(데이터링크)
- 부여 주체 : 제조사
- 변경 가능 여부 : 기본적으로 고정(변경 가능)
- 범위 : 같은 네트워크(LAN) 내 식별

#### IP 주소

- 계층 : OSI 3계층(네트워크)
- 부여 주체 : 네트워크 관리자 또는 DHCP 등
- 변경 가능 여부 : 환경에 따라 변경 가능
- 범위 : 네트워크 전체(라우팅)

### MAC 주소가 사용되는 경우

```text
내 PC가 구글(8.8.8.8)에 접속할 때

1. IP 주소로 목적지 파악 (라우터가 담당)
2. 같은 LAN 안에서는 MAC 주소로 실제 전달
3. ARP로 IP → MAC 주소 변환
```

### ARP

- (Address Resolution Protocol)
- IP주소는 알지만 MAC 주소를 모를 때 사용하는 프로토콜

### 스위치와 MAC의 관계

-스위치는 MAC의 주소 테이블을 직접 관리한다.

```text
포트1 ─── PC-A (AA:AA:AA:AA:AA:AA)
포트2 ─── PC-B (BB:BB:BB:BB:BB:BB)   ← 스위치가 이걸 기억
포트3 ─── PC-C (CC:CC:CC:CC:CC:CC)
```

-> PC-A가 PC-B에게 보내면, 스위치는 포트2로만 전달 (허브처럼 전체 뿌리지 않음)
-> 즉 스위치가 허브보다 효율적임

## Frame

- 데이터 링크 계층에서 데이터를 전송할 때 사용하는 데이터 단위(PDU) 이다
- 계층마다 이름
  - 1 계층 : 비트
  - 2 계층 : 프레임
  - 3 계층 : 패킷
  - 4 계층 : 세그먼트(TCP)/데이터그램(UDP)

###
- 프레임의 구조
  - 목적지 맥 주소 + 소스 MAC 주소 + 데이터
  - 전체 구조
    - 프리앰블 + SFD + 목적지 MAC + 출발지 MAC + 이더타입 + 페이로드 + FCS
```text
  - 각 필드 역할
    - 프리앰블 : 수신 장치가 동기화 줂비하는 신호
    - SFD(Start Frame Delimiter) : 프레임 시작을 알리는 신호
    - 이더타입 : 페이로드가 어떤 프로토콜인지 표시
    - 페이로드 : 실제 데이터
    - FCS(Frame Check Sequence) : CRC-32 방식으로 오류 감지, 오류 있으면 프레임 폐기 
      - CRC-32
        - 데이터를 생성 다항식으로 나눈 나머지로 오류를 검출하는 방식
        - 송신 측 : 데이터 ÷ 생성 다항식 → 나머지(32bit) = FCS에 삽입
        - 수신 측 : (데이터+FCS) ÷ 생성 다항식 → 나머지 = 0이면 정상
        - 오류 검출만 가능, 수정은 불가 (수정은 TCP 몫)
```

## 스위치

- 같은 LAN 안의 장치들을 연결해서 프레임을 목적지에만 선택적으로 전달하는 2계층 장비
- 허브와 달리 스위치는 MAC 주소 테이블을 보고 해당 포트에만 전달한다

### 스위치 동작

1.Learning : 프레임이 들어오면 출발지 MAC 주소 + 포트 번호를 MAC 주소 테이블에 저장하는 것으로 테이블을 채워 나간다.
2.Filtering : 목적지 MAC 테이블에 있으면 해당 포트 외의 나머지 포트로는 전달하지 않는다.
3.Forwarding : 테이블에서 찾은 포트로만 프레임을 전달한다.

| 항목 | 허브 (Hub) | 스위치 (Switch) |
|------|-----------|----------------|
| OSI 계층 | 1계층 (물리) | 2계층 (데이터링크) |
| 전달 방식 | 모든 포트에 전송 | 목적지 포트에만 전송 |
| MAC 테이블 | 없음 | 있음 |
| 충돌 도메인 | 전체가 하나 | 포트마다 분리 |
| 브로드캐스트 도메인 | 전체가 하나 | 전체가 하나 |
| 속도/효율 | 낮음 | 높음 |
| 전이중 통신 | 불가 | 가능 |
| 가격 | 저렴 | 상대적으로 비쌈 |

# WAN

- 멀리 떨어진 LAN들을 연결하는 네트워크크

## 라우터

- 서로 다른 네트워크를 연결해서 패킷을 최적 경로로 전달하는 3계층 장비. IP주소로 연락한다

### 핵심 개념
- Routing Table :  목적지 IP 대역별로 어느 경로로 보낼지 기록한 표
- Next Hop : 목적지까지 한 번에 못가니깐, 다음에 패킷을 전달할 장비의 IP 주소를 가리킨다
- Default Gateway : PC 입장에서 라우터가 기본 게이트웨이이고, 목적지가 다른 네트워크면 일단 라우터한테 보내고, 라우터가 알아서 경로를 찾아준다
- 라우터는 대규모 네트워크에서 트래픽 낭비를 막는 역할도 한다.

## 패킷과 프레임(+)

### 패킷

- IP 프로토콜이 데이터를 전송하기 위해 만드는 3계층 PDU(Protocol Data Unit)
- IP 헤더와 페이로드로 구성

```text
- IP 헤더 구성
  - 출발지 IP 주소
  - 목적지 IP 주소
  - TTL(Time To Live) : 라우터를 거칠 때마다 1씩 감소, 0이 되면 패킷 폐기
```

- 핵심 동작 패킷은 출발지에서 목적지까지 라우터를 몇 개를 거치든 IP 주소가 절대 바뀌지 않음
- 라우터는 목적지 IP를 보고 경로를 결정 (수정 X)

### 프레임

- 이더넷 프로토콜이 같은 네트워크 구간 안에서 패킷을 전달하기 위해 만드는 2계층 PDU
- 이더넷 헤더와 페이로드, FCS로 구성
```text
- 이더넷 헤더 구성
  - 출발지 MAC 주소
  - 목적지 MAC 주소
  - 이더타입
```

- 핵심 동작 프레임음 한 홉 사이에서만 유효함
- 라우터에 도착하면 프레임은 완전히 벗겨지고, 라우터가 라우팅 테이블을 보고 다음 홉 MAC 주소로 새 프레임을 전달

### Hop

- 장치와 장치 사이 한 구간을 홉이라고 한다.
- 라우터 하나를 지나칠 때 마다 1홉

```text
PC A -> 라우터1 -> 라우터2 -> PC B
     1홉       2홉       3홉
```

- 라우터가 많을 수록 홉 수가 늘어난다.

### 패킷 vs 프레임

- 역할이 다르므로 구분 되어있음

```text
- 상위  계층이 하위 계층의 구체적인 구현에 의존하지 않도록 하기 위함
- ip는 하위에 이더넷이 있든, WIFI가 있든, 관 케이블이 있든 신경 쓰지 않음
- 이더넷도 자신이 운반하는 페이로드가 ip인지 ARP인지 신경 쓰지 않음
- 각 계층이 자신의 역할만 수행
```

## 라우팅
