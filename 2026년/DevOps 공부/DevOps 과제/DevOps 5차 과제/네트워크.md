# 🌐 Networking 핵심 정리

---

## 1. 인터넷 / 인트라넷 / 엑스트라넷

| 종류 | 설명 | 접근 범위 |
|------|------|-----------|
| **인터넷 (Internet)** | 전 세계 컴퓨터를 연결하는 공개된 글로벌 네트워크. TCP/IP 기반 | 전체 공개 |
| **인트라넷 (Intranet)** | 기업·기관 내부에서만 사용하는 사설 네트워크. 인터넷 기술(웹·이메일)을 사내 용도로 적용 | 내부 전용 |
| **엑스트라넷 (Extranet)** | 인트라넷을 특정 외부 파트너(협력사·공급업체)에게 일부 공개한 네트워크. 방화벽 + VPN으로 보안 유지 | 제한적 외부 |

> 범위: **인터넷 > 엑스트라넷 > 인트라넷**

---

## 2. LAN (Local Area Network) — 근거리 통신망

같은 건물, 학교, 사무실처럼 **좁은 지역** 내 장치들을 연결하는 네트워크

| 항목 | 내용 |
|------|------|
| 범위 | 수 m ~ 수 km (건물 내 / 캠퍼스 내) |
| 전송 속도 | 10Mbps ~ 10Gbps |
| 소유 | 단일 기관 (사설 소유) |
| 연결 방식 | 유선 이더넷, 무선 Wi-Fi |
| 주요 장비 | 스위치(Switch), 공유기(AP) |

**장점**
- 빠른 속도 / 낮은 지연
- 구성 및 관리 용이
- 비용 저렴

**단점**
- 거리 제한 있음
- 확장 시 추가 장비 필요

---

## 3. WAN (Wide Area Network) — 광역 통신망

도시·국가·대륙 간처럼 **넓은 지역**을 연결하는 네트워크. 인터넷 자체가 대표적 WAN

| 항목 | 내용 |
|------|------|
| 범위 | 국가 ~ 전 세계 |
| 전송 속도 | 수 Kbps ~ 수 Gbps (LAN보다 느림) |
| 소유 | ISP(통신사) 등 다수 조직 |
| 연결 방식 | 광케이블, 위성, 전용선, MPLS |
| 주요 장비 | 라우터(Router) |

**장점**
- 원거리 통신 가능
- 전 세계 연결

**단점**
- LAN보다 느리고 비쌈
- 보안 취약 가능성
- 구성·관리 복잡

> LAN vs WAN: LAN = 교실 내 PC 연결 / WAN = 서울↔부산 지점 연결

## 4. 이더넷 (Ethernet)

IEEE **802.3** 표준. 현재 가장 널리 사용되는 유선 LAN 기술.  
버스형 구조에서 시작 → 현재 **스타형(Star)** 구조로 발전

| 항목 | 내용 |
|------|------|
| 전송 단위 | 프레임(Frame) |
| 주소 체계 | MAC 주소로 장치 식별 |
| 충돌 제어 | CSMA/CD (버스형) / Full-Duplex 환경에서는 충돌 없음 |

**속도 규격**

| 규격 | 속도 | 비고 |
|------|------|------|
| 10BASE-T | 10 Mbps | 꼬임쌍선(UTP) |
| 100BASE-TX | 100 Mbps | Fast Ethernet |
| 1000BASE-T | 1 Gbps | Gigabit Ethernet |
| 10GBASE-T | 10 Gbps | 10G Ethernet |

---

## 5. CSMA/CD

**Carrier Sense Multiple Access / Collision Detection**  
이더넷 버스형 환경에서 충돌을 제어하는 매체 접근 방식

**동작 순서**

1. **CS (반송파 감지)** — 데이터 전송 전 회선이 사용 중인지 확인
2. **MA (다중 접근)** — 회선이 비어있으면 여러 장치가 동시에 전송 시도 가능
3. **CD (충돌 감지)** — 두 장치가 동시에 전송 → 충돌(Collision) 발생 → 즉시 감지
4. **잼 신호(Jam Signal)** 전송 → 모든 장치에 충돌 알림
5. **백오프(Backoff)** — 각 장치가 랜덤한 시간 대기 후 재전송 시도

> Full-Duplex 스위치 환경에서는 CSMA/CD 불필요 — 현재 대부분 환경에서 사용 안 함

## 6. 토큰링 (Token Ring)

IEEE **802.5** 표준. 링(Ring)형 토폴로지에서 **토큰(Token)** 을 순서대로 넘겨받은 장치만 데이터를 전송할 수 있는 방식

| 항목 | 내용 |
|------|------|
| 동작 원리 | 빈 토큰이 링 순환 → 전송할 장치가 토큰 획득 → 데이터 전송 → 토큰 반환 |
| 충돌 | 토큰 소유자만 전송 → **충돌 없음** |
| 속도 | 4Mbps / 16Mbps |

**장점**
- 충돌 없이 안정적
- 트래픽 많을수록 이더넷보다 효율적
- 공정한 접근 (순번 보장)

**단점**
- 한 장치 고장 시 전체 마비
- 토큰 관리 복잡
- 이더넷에 밀려 현재 거의 미사용

## 7. 서브넷 (Subnet)

하나의 큰 네트워크를 **여러 개의 작은 네트워크로 분할**하는 기술.  
**서브넷 마스크(Subnet Mask)** 로 네트워크/호스트 영역을 구분

| 항목 | 내용 |
|------|------|
| 서브넷 마스크 | IP 주소에서 네트워크 부분과 호스트 부분을 구분하는 값. 예) 255.255.255.0 = /24 |
| CIDR 표기 | `192.168.1.0/24` → 앞 24비트 = 네트워크, 나머지 8비트 = 호스트 |
| 서브네팅 목적 | IP 낭비 방지, 보안 강화, 트래픽 분리, 네트워크 성능 향상 |

**예시: `192.168.1.0/24`**

| 구분 | 주소 |
|------|------|
| 네트워크 주소 | 192.168.1.0 |
| 브로드캐스트 | 192.168.1.255 |
| 사용 가능 호스트 | 192.168.1.1 ~ 192.168.1.254 **(254개)** |

## 8. MAC 주소 (MAC Address)

**Media Access Control Address** — NIC(랜카드)에 부여된 **고유한 물리적 주소**

| 항목 | 내용 |
|------|------|
| 형식 | 48비트(6바이트), 16진수 12자리. 예) `AA:BB:CC:DD:EE:FF` |
| 구성 | 앞 24비트: OUI (제조사 식별자) / 뒤 24비트: 장치 고유 번호 |
| OSI 계층 | 2계층 (데이터 링크 계층) |
| IP와 차이 | IP = 논리 주소(변경 가능) / MAC = 물리 주소(하드웨어에 고정) |
| 활용 | LAN 내 장치 식별, 스위치 MAC 주소 테이블, ARP |

> **ARP**: IP 주소 → MAC 주소로 변환해주는 프로토콜 (같은 LAN 내 통신 시 사용)

## 9. DNS (Domain Name System)

도메인 이름(`www.google.com`)을 **IP 주소로 변환**해주는 분산 데이터베이스 시스템.  
인터넷의 **'전화번호부'**

**조회 과정**

1. 사용자가 브라우저에 `www.google.com` 입력
2. **로컬 캐시** 확인 → 없으면 로컬 DNS 서버(ISP) 조회
3. **Root DNS** → `.com` TLD DNS → 구글 권한 DNS 순서로 조회
4. IP 주소(예: `142.250.196.68`) 반환 → 캐시 저장
5. 해당 IP로 접속

| 레코드 | 역할 |
|--------|------|
| A | 도메인 → IPv4 주소 |
| AAAA | 도메인 → IPv6 주소 |
| MX | 메일 서버 지정 |
| CNAME | 도메인 별칭(Alias) |
| NS | 네임서버 지정 |

| 항목 | 내용 |
|------|------|
| 포트 | UDP/TCP **53번** |
| TTL | DNS 캐시 유효 시간 (만료 후 재조회) |

## 10. 클라우드 컴퓨팅 (Cloud Computing)

인터넷을 통해 서버·스토리지·DB·SW 등 **IT 자원을 서비스 형태로 제공**받는 컴퓨팅 방식

**서비스 모델**

| 모델 | 의미 | 예시 |
|------|------|------|
| **IaaS** | Infrastructure as a Service — 가상 서버, 스토리지 등 인프라 제공 | AWS EC2, Azure VM |
| **PaaS** | Platform as a Service — 개발·배포 플랫폼 제공. OS·미들웨어 관리 불필요 | Heroku, GCP App Engine |
| **SaaS** | Software as a Service — 완성된 SW를 구독 형태로 제공 | Gmail, Office 365 |

**배포 모델**

| 모델 | 설명 |
|------|------|
| 퍼블릭 클라우드 | AWS, Azure, GCP 등 불특정 다수에게 공개 제공 |
| 프라이빗 클라우드 | 특정 기업·기관 전용. 보안 높음 |
| 하이브리드 클라우드 | 퍼블릭 + 프라이빗 혼합 운영 |

**장점**
- 초기 비용 절감 (CAPEX → OPEX)
- 유연한 확장/축소 (탄력성)
- 어디서나 접근 가능
- 자동 백업 및 재해 복구

**단점**
- 인터넷 의존성 (장애 시 마비)
- 데이터 보안·개인정보 우려
- 장기 사용 시 비용 증가
- 벤더 종속(Lock-in) 위험

> DevOps 핵심 환경 — AWS, GCP, Azure 기반 CI/CD, Docker/Kubernetes 운영이 표준
