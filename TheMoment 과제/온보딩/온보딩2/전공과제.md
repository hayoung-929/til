# AWS (Amazon Web Services)

---

## 1. AWS란?

**Amazon이 제공하는 클라우드 컴퓨팅 플랫폼**이다.

서버, 스토리지, 네트워크, 데이터베이스 등 IT 인프라를 인터넷을 통해 서비스 형태로 제공한다. 직접 서버를 구매·운영하지 않고 필요한 만큼 빌려서 사용하는 방식이다.

> 현재 전 세계 클라우드 시장 점유율 1위 (약 32%)이며, GCP(Google), Azure(Microsoft)와 함께 3대 퍼블릭 클라우드로 꼽힌다.

---

## 2. 글로벌 인프라

### 리전 (Region)
AWS 데이터센터가 위치한 지리적 구역이다. 전 세계 30개 이상의 리전이 있으며, 서울 리전은 `ap-northeast-2`다. 서비스를 배포할 때 어느 리전에 올릴지 선택해야 한다.

> 사용자와 가까운 리전을 선택할수록 응답 속도(레이턴시)가 낮아진다.

### 가용 영역 (AZ, Availability Zone)
하나의 리전 안에 있는 독립적인 데이터센터 클러스터다. 서울 리전은 4개의 AZ로 구성된다. 하나의 AZ에 장애가 발생해도 다른 AZ는 영향을 받지 않아 고가용성을 확보할 수 있다.

### 엣지 로케이션 (Edge Location)
콘텐츠를 사용자에게 빠르게 전달하기 위한 캐시 서버 거점이다. CloudFront(CDN)가 엣지 로케이션을 통해 동작한다.

## 3. 핵심 서비스

### EC2 (Elastic Compute Cloud)
**가상 서버를 제공하는 서비스**다.

원하는 CPU, RAM, OS를 선택해서 가상 서버(인스턴스)를 생성할 수 있다. 필요할 때 켜고 끌 수 있으며, 사용한 시간만큼 요금을 낸다.

- 인스턴스 타입: t2.micro (소규모), m5.large (범용), c5.xlarge (컴퓨팅 최적화) 등
- AMI (Amazon Machine Image): 인스턴스를 시작할 때 사용하는 OS 이미지

### S3 (Simple Storage Service)
**오브젝트 스토리지 서비스**다.

파일을 버킷(Bucket)이라는 공간에 업로드해서 저장한다. 용량 제한이 없고 파일 하나당 최대 5TB까지 저장할 수 있다. 이미지, 영상, 로그, 백업 파일 등 정적 파일 저장에 주로 사용한다.

- 버킷: S3에서 파일을 담는 최상위 컨테이너
- 객체: 버킷 안에 저장되는 파일 단위

### RDS (Relational Database Service)
**관리형 관계형 데이터베이스 서비스**다.

MySQL, PostgreSQL, MariaDB, Oracle 등을 AWS가 직접 운영해준다. 백업, 패치, 복제 등을 자동으로 처리해줘서 개발자는 쿼리 작성에만 집중할 수 있다.

### VPC (Virtual Private Cloud)
**AWS 안에서 논리적으로 격리된 가상 네트워크 공간**이다.

EC2, RDS 등 AWS 자원을 배치하는 네트워크 환경을 직접 설계할 수 있다. 서브넷, 라우팅 테이블, 인터넷 게이트웨이 등을 설정해서 네트워크 구조를 구성한다.

### IAM (Identity and Access Management)
**AWS 자원에 대한 접근 권한을 관리하는 서비스**다.

누가, 어떤 자원에, 어떤 작업을 할 수 있는지 정의한다. 루트 계정 대신 IAM 사용자를 만들어서 최소 권한만 부여하는 것이 보안 기본 원칙이다.

- 사용자 (User): 개별 접근 주체
- 그룹 (Group): 사용자를 묶어서 권한을 일괄 적용
- 역할 (Role): EC2 같은 서비스에 권한을 부여할 때 사용
- 정책 (Policy): 허용·거부할 작업을 JSON으로 정의

### ELB (Elastic Load Balancing)
**트래픽을 여러 서버에 자동으로 분산시키는 서비스**다.

요청이 몰려도 한 서버에 집중되지 않도록 여러 EC2 인스턴스에 고르게 나눠준다. 서버 중 하나가 다운돼도 다른 서버로 자동 전환된다.

### Auto Scaling
**트래픽에 따라 EC2 인스턴스 수를 자동으로 늘리고 줄이는 서비스**다.

CPU 사용률 같은 지표가 임계값을 넘으면 인스턴스를 자동으로 추가(Scale Out)하고, 부하가 줄면 다시 제거(Scale In)한다. ELB와 함께 사용하면 고가용성 아키텍처를 구성할 수 있다.

### CloudWatch
**AWS 자원의 모니터링 및 로깅 서비스**다.

EC2의 CPU 사용률, 네트워크 트래픽, 애플리케이션 로그 등을 수집하고 시각화한다. 특정 임계값 초과 시 알림(Alarm)을 보내거나 Auto Scaling을 트리거할 수 있다.

### Lambda
**서버 없이 코드를 실행하는 서버리스 서비스**다.

서버를 프로비저닝하지 않고 함수 단위로 코드를 올리면 이벤트 발생 시 자동으로 실행된다. 실행 시간과 횟수만큼만 요금을 낸다.

## 4. 요금 구조

AWS는 **사용한 만큼 내는 종량제(Pay-as-you-go)** 방식이다.

### 주요 과금 기준

- **EC2**: 인스턴스 유형 × 실행 시간 (시간 단위)
- **S3**: 저장 용량 (GB) + 요청 횟수 + 데이터 전송량
- **RDS**: 인스턴스 유형 × 실행 시간 + 스토리지 용량
- **데이터 전송**: AWS 내부 → 외부 인터넷으로 나가는 트래픽에 과금 (들어오는 건 무료)

### EC2 구매 옵션

- **온디맨드 (On-Demand)**: 필요할 때 켜고 끄는 기본 방식. 가장 비쌈
- **예약 인스턴스 (Reserved)**: 1년 또는 3년 약정 시 온디맨드 대비 최대 72% 할인
- **스팟 인스턴스 (Spot)**: AWS의 남는 자원을 경매 방식으로 사용. 최대 90% 저렴하지만 언제든 회수될 수 있음
- **Savings Plans**: 특정 사용량을 약정하면 할인 적용

### 프리 티어 (Free Tier)
신규 가입 후 12개월간 일부 서비스를 무료로 사용할 수 있다.

- EC2 t2.micro: 월 750시간
- S3: 월 5GB
- RDS: 월 750시간 (db.t2.micro)
